Programming in Ocaml

Ocaml is a high-level, yet highly efficient programming language which supports both object-oriented and functional programming styles. Ocaml has a number of attributes which make it very well-suited to writing a network simulator:
  • Speed, speed, speed..
  • An interactive toplevel, which natively gives the advantages of a scripting language (interactively develop scripts, examine simulator state). This makes things much simpler than integrating a scripting language into a non-scripted language (say c++ and otcl), since everything remains in one language, with no bindings to write, etc.
  • A replay debugger.
  • Support for high-level, structured data types. For example a data type representing an AODV route discovery (sequence of hops + possible route repair floods) can be defined in a few lines.

Documentation

We recommend having at least basic familiarity with ocaml (or another ML-derived language) prior to starting out in nab. There are many great resources (Books and Tutorials) available to learn ocaml; two excellent ones in particular are:
  • The 'Oreilly book' (online in English or in French) is an well-written and well-structured book, with both introductory and advanced material. (a good read through chapters 2, 3, and 4 should be sufficient to get started in nab).
  • A more practical approach is in R. Jones' tutorial, which teaches ocaml by comparing and contrasting it with c and java.

Community

A lot of valuable information can be obtained through the ocaml mailing lists.

ocaml_beginners is a great place to ask questions when starting out, and the caml-list is also a valuable resource for less trivial problems.
