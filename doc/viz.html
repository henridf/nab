Comparison of three last encounter routing protocols.
A route is computed with each protocol for the same source-destination pair.
(source is at upper right corner, destination at bottom).
The network has 4000 nodes which have been moving with a billiard mobility model.

Grat. RREP Local Repair
AODV routes in a 2000 node network.
Destination is in upper left corner.

The darker line represents the effective route taken by a packet.
The red lines represent a route request (RREQ) flood necessary to route the packet. Expanding ring search is used.
In the first case, the route request is answered by an intermediate node quite close to the source, hence the flood is local around the source.
In the second case, the packet advances along a pre-existing route until it hits a broken link in the middle; a local repair is performed at that point.
